Я написал бэкенд на половину. Сервер на PostgreSQL. 
Сон с колонки сохраняется в db через POST запрос: http://our_server_name.ru/api/dream
с телом в виде JSON в формате { "text": "распознанный текст сна", "deviceID": "SberBoomSerialNumber" }
Каждый сон привязывается к серийному номеру колонки. Вроде, серийный номер колонки можно считать программно.
Колонка сможет сама отправлять такие запросы после распознавания текста?

Как будет работать:
1. Пользователь регистрируется на сайте с серийным номером своей колонки.
2. После этого можно записывать сны с колонки. При POST запросе с колонки будет проверяться, 
   введен ли серийный номер колонки (deviceID) у какого-то пользователя, если введен,
   то сон будет записан. Иначе - запрос отклоняется (чтобы враги без регистрации не забили базу хламом).
3. Пользователь сможет просматривать список снов по датам, открывать и удалять нужный сон.

Вопросы:
Сны могут быть длинными? 
Колонка будет инициировать начало записи сна и окончание, и результат придет в бэкенд в виде целого сна?
Или сон будет записан в базе кусками и надо будет его как-то склеивать (близлежащие куски по времени)? 
Наверное, в списке снов надо выводить не весь сон, а только его начало, снов может быть много 
и выводить сразу много текста не красиво.
Или предлагай что-то еще.

В идеале выложить бэкенд где-то с доступом из-вне, чтобы можно было одновременно править его и тестировать.
